Git, GitHub və Deploy Necə Olmalıdır?

Layihənizi koddan canlı sayta qədər aparmağın addım-addım izahı

Başla ↓
01

Giriş

📚

Git Nədir?

Git — versiya idarəetmə sistemidir. Kodunuzun tarixçəsini saxlayır, dəyişiklikləri izləyir və komanda ilə birgə işləməyə imkan verir.

☁️

GitHub Nədir?

GitHub — Git repozitoriyalarınızı onlayn saxlamaq üçün platformadır. Kod paylaşmaq, əməkdaşlıq etmək və layihələri idarə etmək üçün istifadə olunur.

🚀

Deploy Nədir?

Deploy — layihənizi serverdə yerləşdirmək və istifadəçilər üçün əlçatan etmək prosesidir. GitHub Pages, Netlify, Vercel kimi xidmətlər bunu asanlaşdırır.

02

Git ilə İşləmək

1

Git-i quraşdırın

git-scm.com saytından yükləyin və quraşdırın. Sonra terminaldə yoxlayın:

git --version
2

Konfiqurasiya edin

Git-ə adınızı və e-poçtunuzu bildirin:

git config --global user.name "Adınız" git config --global user.email "email@example.com"
3

Repo yaradın

Layihə qovluğunda Git-i başladın:

git init
4

Dəyişiklikləri qeyd edin

Faylları izləməyə əlavə edin və commit edin:

git add . git commit -m "İlk commit"
03

GitHub-a Yükləmək

1

GitHub hesabı yaradın

github.com saytında qeydiyyatdan keçin.

2

Yeni repo yaradın

GitHub-da "New repository" düyməsini basın, ad verin və yaradın. README əlavə etməyin.

3

Lokal repo-nu bağlayın və push edin

GitHub-ın verdiyi əmrləri terminaldə icra edin:

git remote add origin https://github.com/istifadeci/repo.git git branch -M main git push -u origin main
04

Deploy Üsulları

Ən Asan

GitHub Pages

Statik saytlar üçün pulsuz hosting. Birbaşa GitHub repo-dan deploy edir.

  1. GitHub-da repo-nun Settings bölməsinə keçin
  2. Pages seçin
  3. Source olaraq main branch seçin
  4. Save basın
  5. Saytınız username.github.io/repo ünvanında hazırdır!

Netlify

Avtomatik deploy, form idarəetmə və serverless funksiyalar.

  1. netlify.com-da GitHub ilə daxil olun
  2. "New site from Git" seçin
  3. Repo-nu seçin və deploy edin

Vercel

Next.js və digər framework-lər üçün ideal. Çox sürətli.

  1. vercel.com-da GitHub ilə daxil olun
  2. Repo-nu import edin
  3. Avtomatik konfiqurasiya ilə deploy edin
05

Faydalı Git Əmrləri

Əmr İzah
git status Dəyişikliklərin vəziyyətini göstərir
git log --oneline Commit tarixçəsini qısa göstərir
git branch Branch-ları siyahılayır
git checkout -b yeni-branch Yeni branch yaradıb keçir
git merge branch-adi Branch-ı birləşdirir
git pull Uzaq repo-dan dəyişiklikləri çəkir
git clone url Repo-nu yerli kompüterə klonlayır
git stash Dəyişiklikləri müvəqqəti saxlayır
git diff Dəyişikliklərin fərqini göstərir
git reset --soft HEAD~1 Son commit-i geri qaytarır (dəyişikliklər qalır)
